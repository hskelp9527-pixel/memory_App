# Memory Vine UI/UX 改版设计

## 目标

本次改版不增加新的业务能力，专注于把已有 MVP 的交互从“可用”提升到“像一本真实的复古记忆手账”。

重点修改四处：
- 新增记忆弹窗
- 记忆藤蔓时间轴
- 新增按钮
- 登录后的封面进入方式

## 1. 新增记忆弹窗

### 现状问题
- 左侧上传区域像后台表单，不像贴照片
- “媒体文件”标题、上传提示文案、文件按钮噪声太多
- 日期与时间输入缺乏可选性，交互半成品

### 改版方案
- 左侧整块相片槽作为唯一上传入口
- 去掉“媒体文件”标题、去掉上传提示文案、去掉原生文件按钮外观
- 点击左侧相片槽任意位置直接触发文件选择
- 选中文件后在左侧固定尺寸相框内立即显示预览
- 再次点击预览区可替换图片/视频
- 日期字段使用原生 `date`
- 时间字段使用原生 `time`，24 小时制

### 体验预期
- 交互更像“贴一张照片”
- 表单视觉噪声显著减少
- 用户不需要理解“先选文件再填表单”的技术流程

## 2. 记忆藤蔓时间轴

### 现状问题
- 页面中心缺少真正的“藤蔓”视觉骨架
- 卡片只是左右交错，概念上还不是“蔓生”

### 改版方案
- 页面中心增加一条竖向藤蔓主线
- 主线上点缀少量叶片节点
- 卡片继续左右交错，但与中线建立固定锚点关系
- 中线在移动端缩弱，避免压迫单列布局

### 体验预期
- “记忆之蔓”从命名变成可感知的视觉结构
- 时间轴拥有更强的叙事感和纵向浏览引导

## 3. 新增按钮

### 现状问题
- 当前按钮只是一个普通圆形加号，廉价且不符合整体气质

### 改版方案
- 改成黄铜/邮戳质感的悬浮按钮
- 视觉重心从符号 `+` 改为更温和的“羽毛笔 + 加号”组合或压印式 `+`
- hover 有轻微浮起
- active 有印章按压感

### 体验预期
- 功能入口不再突兀
- 视觉语言与整体复古材质统一

## 4. 封面进入方式

### 现状问题
- “打开锁扣”按钮过于直接，像 CTA，不像翻开一本书

### 改版方案
- 整本书作为点击区域
- 不再显示独立按钮
- 点击后执行短时翻书动画
- 动画完成后自动跳转到 `/timeline`
- 保留极弱提示，不抢视觉中心

### 体验预期
- 从“点按钮进入下一页”升级为“翻开记忆”
- 登录后的第一步更有仪式感

## 5. 实施原则

- 不改数据库结构
- 不改 API 契约
- 优先改交互与视觉，不扩大业务范围
- 保持移动端可用性

## 6. 移动端补充决议

- 时间轴移动端采用“细长日记流”而非桌面缩放版
- 卡片在手机上进一步缩窄与缩短，围绕藤蔓中线轻微错位
- 新建记忆弹窗在手机上保持居中弹窗，但首屏必须同时看见小相片槽和表单起始字段
