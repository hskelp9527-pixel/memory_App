## PRD 关键信息
- 产品定位：复古、私密、个人记忆手账 Web 应用，响应式兼容移动端。
- 核心流程：注册/登录 -> 手账封面 -> 时间轴浏览 -> 新增记忆 -> 详情弹窗。
- 数据主体：`memories` 表 + `memories-media` 存储桶。
- 安全模型：以 `user_id` + Supabase RLS 保证私密隔离。

## 当前观察
- 样式稿明显偏强视觉表达，适合保留为设计基线。
- PRD 原始技术方案偏“纯静态 + Supabase 直连”，开发快，但在路由、组件复用、状态管理、后续维护上会较弱。
- 登录页与注册页的视觉语言基本统一，但注册页样式稿仍保留“邮箱”字段，和 PRD 的“用户名+密码”方案不一致，后续应以隐式邮箱方案收敛。
- 新增弹窗和详情弹窗都已具备可映射的 UI 容器，适合直接抽象成复用组件。
- 时间轴页面存在超出 PRD 的底部导航/探索型入口，这更像设计探索稿，不一定进入 MVP。
