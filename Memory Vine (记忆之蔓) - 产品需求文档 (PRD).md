1. 产品概述产品名称：Memory Vine (记忆之蔓)产品愿景：打造一个极具复古质感、高度私密的个人数字化记忆手账。让用户像翻阅老旧相册和日记本一样，珍藏和回味生命中的重要瞬间。目标用户：注重隐私、喜爱复古手账风格的个人用户（极小规模用户量，主要为开发者本人及少数密友）。核心终端：Web 浏览器端（主导），采用响应式布局完美兼容移动端设备展示。核心特性：极简“用户名+密码”无痕注册/登录。完全的数据隔离，每个人拥有100%私密的时间轴。支持图片与视频等多媒体格式。基于时间轴的“拍立得”式沉浸浏览体验。支持高度还原的复古“邮戳”细节（地点与时间）。2. 技术架构选型本项目采用 Serverless (无服务器) + BaaS (后端即服务) 的轻量化架构：前端：原生 HTML5 + CSS3 + 原生 JavaScript (Vanilla JS)。无需 React/Vue 等复杂框架，无需打包工具。后端与数据库：Supabase (开源 Firebase 替代品)。认证 (Auth)：使用 Supabase Auth。数据库 (Database)：使用 Supabase PostgreSQL。对象存储 (Storage)：使用 Supabase Storage 存储图片和视频。前后端交互：前端通过引入 Supabase 官方提供的 JS SDK CDN 链接，直接在浏览器端调用 API 实现数据的增删改查。3. 核心业务流程 (User Flow)鉴权访问：用户访问网站 -> 展示【图1：登录页】 -> 若无账号，切换至【图2：注册页】输入用户名和密码直接注册并自动登录 -> 进入首页。开启手账：登录成功 -> 展示【图5：手账封面】 -> 点击封面锁扣交互区 -> 路由跳转/动画过渡至个人私密时间轴。时间轴浏览：进入【图3：时间轴页】 -> 页面初始化时通过 JS SDK 请求当前登录用户的记忆列表 -> 渲染带图文的拍立得卡片列表（按时间倒序或正序排列）。记录瞬间：点击右侧悬浮【+按钮】 -> 弹出【图4：新增记忆弹窗】 -> 填写内容并上传媒体文件 -> 提交保存至 Supabase -> 成功后自动关闭弹窗并刷新时间轴。重温记忆：在时间轴点击某张记忆卡片 -> 弹出【图6：记忆详情弹窗】 -> 展示多媒体大图/视频、复古邮戳（若填写了地点时间）、详细正文。4. 功能模块详细说明4.1 账号与鉴权模块 (对应图1、图2)功能描述：提供极简的用户准入机制。交互逻辑：登录 (图1)：输入“用户名”与“密码”。注册 (图2)：输入“用户名”、“密码”及“确认密码”。技术实现 (隐式邮箱方案)：因 Supabase 默认需邮箱注册，前端在捕获到用户输入的 username 后，JS 自动拼接虚拟后缀（如 ${username}@memoryvine.local）作为内部账号提交给 Supabase 接口。用户全程无感知。注册成功后直接调用登录方法，将 JWT Token 存入本地 localStorage，实现持久化登录。4.2 档案封面模块 (对应图5)功能描述：氛围感引导页。交互逻辑：校验用户是否已登录，未登录则重定向回登录页。页面居中展示带有“MEMORIES - Our Story”的皮质手账封面。用户点击“锁扣 (Clasp)”区域，触发开启事件，进入时间轴主页 (图3)。4.3 记忆时间轴模块 (对应图3)功能描述：私密日记的列表展示。交互逻辑：根据当前用户的 user_id 从数据库拉取属于该用户的所有记忆记录。左、右交错渲染拍立得卡片（移动端自适应为单列向下）。卡片元素包含：媒体缩略图（若为视频则截取第一帧或展示统一封面）、手写体配文 (caption)、日期 (memory_date)。右侧常驻一个悬浮的【+按钮】，用于触发新增功能。4.4 新增记忆模块 (对应图4)功能描述：发布新的日记内容。表单字段说明：media (媒体文件)：必填。允许选择图片 (image/*) 或视频 (video/*)。title (标题)：必填。记忆的标题。date (发生日期)：必填。日历组件选择器。caption (一句话配文)：必填。用于时间轴拍立得卡片底部展示。content (详细正文)：必填。记录详细心绪的长文本区域。location (地点)：可选。记录发生的城市/地点（用于图6的邮戳）。specific_time (具体时间)：可选。如 "14:32"（用于图6的邮戳）。技术实现 (提交逻辑)：前端校验必填项。调用 Supabase Storage API，将媒体文件上传至 memories-media 存储桶。获取上传成功后的文件公共直链 (Public URL)。将表单文本数据连同媒体 URL、媒体类型 (image或video) 一起插入 Supabase memories 数据库表。提示成功，关闭弹窗，重载图3列表。4.5 记忆详情模块 (对应图6)功能描述：展示单条记忆的完整信息。交互逻辑：媒体展示区：根据 media_type 判断。如果是 image，使用 <img> 标签渲染；如果是 video，使用 <video controls> 标签渲染，并在UI层覆盖图6设计稿中的播放(Play)按钮图标。复古邮戳区 (右上角)：若该条数据包含 location 和 specific_time，则将参数填入对应的红色邮戳 UI 中渲染。若未填写可选数据，邮戳区域可显示默认占位符（如“未知地点”），或直接通过 CSS 隐藏该邮戳元素。文字内容区：展示标题、正文文本。右上角提供【关闭 (X)】按钮，点击后弹窗消失，返回时间轴当前位置。5. 数据库结构设计 (Supabase Schema)需要在 Supabase 的 SQL Editor 中执行建表与权限配置。5.1 数据表: memories字段名 (Column)数据类型 (Type)约束 (Constraints)描述 (Description)iduuidPrimary Key, Default uuid_generate_v4()记忆记录的唯一标识符user_iduuidForeign Key (auth.users.id), Not Null记录所有者，实现私密隔离的核心titletextNot Null日记的标题memory_datedateNot Null记忆发生的日期 (如 1998-10-24)specific_timetextNullable[可选] 记忆发生的具体时间 (如 14:32)locationtextNullable[可选] 记忆发生的地点 (如 京都)media_urltextNot Null媒体文件在 Storage 中的访问链接media_typetextNot Null媒体类型，限定值为 'image' 或 'video'captiontextNot Null拍立得照片底部的一句话配文contenttextNot Null详细的日记正文created_attimestampDefault now()数据真实创建/入库的时间5.2 存储桶设计 (Storage Bucket)Bucket Name: memories-media权限:上传 (Insert): 仅限已登录用户 (Authenticated)。读取 (Select): 仅允许用户读取自己上传的文件 (基于 RLS 策略，或为了简单起见设置为公共可读，因为 URL 包含难以猜测的 UUID 且数据库表级已做了读隔离)。5.3 数据安全策略 (Row Level Security - RLS)为了保证“每个注册的人有自己的独立的私密时间轴”，必须在 Supabase 开启 RLS： code SQLdownloadcontent_copyexpand_less-- 启用表的 RLS
ALTER TABLE memories ENABLE ROW LEVEL SECURITY;

-- 策略 1: 用户只能查询 (SELECT) 自己的记忆
CREATE POLICY "Users can view their own memories" ON memories
    FOR SELECT USING (auth.uid() = user_id);

-- 策略 2: 用户只能插入 (INSERT) 属于自己的记忆
CREATE POLICY "Users can insert their own memories" ON memories
    FOR INSERT WITH CHECK (auth.uid() = user_id);

-- 策略 3: 用户只能修改 (UPDATE) 自己的记忆
CREATE POLICY "Users can update their own memories" ON memories
    FOR UPDATE USING (auth.uid() = user_id);

-- 策略 4: 用户只能删除 (DELETE) 自己的记忆
CREATE POLICY "Users can delete their own memories" ON memories
    FOR DELETE USING (auth.uid() = user_id);6. 后续开发建议 / Next Steps目前 PRD 已梳理完毕，你可以按照以下步骤开启你的开发工作：准备环境：前往 Supabase 官网 注册免费账号，新建一个 Project。执行建表：在左侧菜单的 "SQL Editor" 中，运行第 5.1 和 5.3 节的建表和 RLS 策略语句。新建存储桶：在 "Storage" 菜单中，新建一个名为 memories-media 的 Bucket，并配置允许上传图片和视频。获取对接密钥：在项目设置 (Project Settings) -> API 中，拿到你的 Project URL 和 anon public key。前端接入：在你的 HTML 文件头部引入：<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>初始化 Supabase 客户端： code JavaScriptdownloadcontent_copyexpand_lessconst supabaseUrl = 'YOUR_PROJECT_URL';
